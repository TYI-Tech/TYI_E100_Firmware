# 设备配置

该固件仓库的设计目标是让每台机型只维护一个可编辑文件：

- `machine.env`

## 常见需要修改的参数

- `FCU_DEVICE`
  PX4 与机载电脑通信的串口设备，例如 `/dev/ttyTHS1`
- `FCU_BAUD`
  PX4 串口波特率
- `MID360_BD_LIST`
  Livox 驱动使用的 MID360 序列号
- `MID360_SN_SUFFIX`
  MID360 序列号最后两位，用于推导 `192.168.1.(100 + 后两位)`
- `MID360_LIDAR_IP`
  如果不使用默认推导规则，可在这里直接指定 MID360 IP
- `HOST_ETH_IFACE`
  连接 MID360 的宿主机有线网口
- `HOST_ETH_IP`
  宿主机用于 MID360 通信的 IP，应按现场网络规划填写，例如 `192.168.1.50` 或 `192.168.1.5`

## `configure_machine.sh` 会改哪些内容

执行 `bash ./scripts/deploy.sh` 时，会调用 `scripts/configure_machine.sh` 并完成以下更新：

- `.env`
  写入 `FCU_URL` 与 `LIVOX_BD_LIST`
- `configs/livox/MID360_config.json`
  写入宿主机 IP、MID360 IP 和对应 UDP 端口
- 宿主机网卡配置
  将连接 MID360 的网口设置为目标 IP

## 说明

- 日常使用只需要关注 `machine.env`，除非有特殊部署需求，一般不需要手动修改 `.env`。
- `configure_machine.sh` 会根据 `machine.env` 自动回写 `.env` 与 `MID360_config.json`，避免重复修改多处文件。
- 如果后续接入 `TYI_Plugin_Ctl`，LIO 输入话题和参考坐标系仍可通过 `configs/fastlio_to_mavros/bridge.yaml` 调整。
