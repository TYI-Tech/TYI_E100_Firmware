# 快速上手

该仓库面向 Ubuntu 20.04 + Docker + Docker Compose 的产品固件部署环境。
正常情况下，只需要修改一个文件：`machine.env`。

## 首次部署

```bash
git clone git@github.com:TYI-Tech/TYI_E100_Firmware.git
cd TYI_E100_Firmware
bash ./scripts/check_host.sh
vim machine.env
bash ./scripts/deploy.sh
```

`deploy.sh` 会按顺序完成三件事：

- 应用当前机型的 UART、MID360 与宿主机网卡配置
- 基于本地源码构建 Docker 镜像
- 以后台方式启动运行容器

`check_host.sh` 会在部署前检查 Docker、Docker Compose 以及关键配置文件是否齐备。

执行 `deploy.sh` 后，预期结果为：

- 本地源码会被构建成 Docker 镜像
- `base-stack` 服务会以后台方式启动
- 可通过 `status.sh`、`logs.sh`、`enter.sh` 继续查看运行状态

补充说明：

- 当前版本已经完成实机 fresh build 验证，可直接从源码构建
- 构建阶段已内置 GeographicLib 所需 geoid 文件，不再依赖临时在线下载该资源
- 如遇到瞬时网络抖动，镜像构建阶段会自动重试基础 `apt` 安装

## 部署完成后

日常使用可执行以下命令：

```bash
bash ./scripts/status.sh
bash ./scripts/logs.sh
bash ./scripts/enter.sh
```

如果目标机器上的 Docker 需要管理员权限，脚本会自动回退到 `sudo docker compose`。
