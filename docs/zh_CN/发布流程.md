# 发布流程

以下流程用于 `TYI E100` 固件仓库的版本发布。

## 建议顺序

1. 完成代码、配置与文档更新
2. 更新 [VERSION](../../VERSION)
3. 更新 [更新记录](../../CHANGELOG.md)
4. 更新 [版本说明](版本说明.md)
5. 执行发布检查：

```bash
bash ./scripts/release_check.sh
```

6. 推送到远端仓库
7. 创建 Git 标签，例如：

```bash
git tag v0.1.1
git push origin v0.1.1
```

8. 在 GitHub Release 页面发布版本说明

## GitHub Release 建议内容

- 版本号
- 主要变更
- 构建或部署影响
- 已完成的实机验证范围
- 是否需要用户修改 `machine.env`

## 本仓库当前建议

- 对运行逻辑变更，优先补充 `CHANGELOG.md` 与 `版本说明.md`
- 对部署链路变更，优先补充 `快速上手.md`、`使用说明.md`、`常见问题.md`
- 发布前至少执行一次 `bash ./scripts/release_check.sh`
