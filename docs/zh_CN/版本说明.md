# 版本说明

## 0.1.2

本版本在 `0.1.1` 的基础上，进一步补齐了预编译镜像与源码构建并行支持的部署方式，并同步发布了 `0.1.2` 预编译镜像标签。

主要更新：

- 默认部署路径改为直接拉取阿里云 ACR 预编译镜像
- 保留本地源码构建路径，可通过 `bash ./scripts/deploy.sh --build` 启用
- 新增 `docker-compose.build.yml`，用于叠加本地构建配置
- 更新部署与运维脚本，使其同时支持拉镜像模式和源码构建模式
- 完善中英文说明文档，补充 ACR 登录、镜像拉取与构建回退路径
- 已在 `uav-nx` 上验证默认镜像拉取链路可正常登录 ACR 并拉取已发布镜像
- 已同步发布 ACR `0.1.2`、`latest` 与 `stable` 标签

适用结论：

- 用户现在可以优先使用预编译镜像快速部署
- 如无镜像仓库访问权限，仍可直接使用源码构建模式
- 当前默认预编译镜像版本为 `0.1.2`，运行链路保持不变

## 0.1.1

本版本完成了面向产品固件仓库的源码构建闭环验证，并已在 `uav-nx` 板卡上完成实机验证。

主要更新：

- 支持从源码直接执行 `docker compose build --no-cache base-stack`
- 在镜像构建阶段自动规范 `livox_ros_driver2` 的 ROS1 清单文件，确保 fresh build 可识别工作区包
- 将 GeographicLib 所需的 `egm96-5` geoid 数据随仓库提供，避免额外在线下载该资源
- 为基础 `apt` 依赖安装增加自动重试，降低瞬时 DNS 或镜像源波动的影响
- 已对运行链路完成验证：
  `livox_lidar_publisher2`、`robot/dlio_odom`、`fastlio_to_mavros`、`mavros`
- 已对关键话题完成验证：
  `/robot/dlio/odom_node/odom`、`/mavros/vision_pose/pose`

适用结论：

- 用户现在可以拉取本仓库源码后，直接通过 `docker compose` 完成构建与运行
- 机型差异仍建议仅通过 `machine.env` 管理
- 如需控制桥接能力，继续通过独立的 `TYI_Plugin_Ctl` 包提供
